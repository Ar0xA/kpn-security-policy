Rationale Nature of Public Cloud
Description The confidentiality, integrity and availability security requirements for data
classified as Internal use allow it to be stored on a cloud infrastructure that is
provisioned for open use by the public. It is owned, managed and operated by a
third party, and it exists on the premises of the cloud provider.

Rationale Nature of Private Cloud
Description The confidentiality, integrity and availability security requirements for data
classified as Confidential allow it to be stored on a cloud infrastructure that is
installed for exclusive use by KPN. It is owned, managed and operated by a
third party or in combination with KPN, and it exists on premise or off premise.
The cloud infrastructure is not a private cloud when a public cloud solution is
required to operate it.

Rationale KPN Data Center
Description The confidentiality, integrity and availability security requirements for data used
for the continuous delivery of Critical or Vital services, information on KPN's
vulnerabilities and for data classified as Secret only allow it to be stored on an
infrastructure that is exclusively used by KPN, owned by KPN and installed on
KPN premise.
Compliance with internal policies (KSP security measures) is required.

Requirement Perform an (information) security risk assessment
Description Prior to purchasing or using cloud services an (information) security risk
assessment must be performed, which takes into account:
the type, classification and importance of information that may be handled in the
cloud (e.g., commercial information, financial information, intellectual property
(IP), legal, regulatory and privileged information (LRP), logistical information,
management information or personally identifiable information (PII)).
